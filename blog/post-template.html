Building a portfolio that doesn't just look like a template

March 2, 2026 • 5 min read

When setting out to build a personal developer portfolio, the easiest path is usually grabbing a standard Next.js or Astro template and modifying the colors slightly. However, if you want to stand out, you need to inject some personality.

The Power of Interactivity

One way to keep people on your site is to give them something to do. Instead of just listing my skills in a boring grid, I decided to include a fully functional game of 2048 right on the homepage.

"The best portfolios aren't just read; they are experienced."

Code Example

Keeping the UI logic separated from your data is key. Here's roughly how you might structure the grid logic:

function restartGame() {
    grid = Array(SIZE).fill().map(() => Array(SIZE).fill(null));
    score = 0;
    addNewTile(true);
    renderBoard();
}

Using vanilla HTML/CSS/JS means zero build steps, and publishing to GitHub Pages is as easy as running a git commit push. Sometimes, the simplest solutions are the most elegant.
